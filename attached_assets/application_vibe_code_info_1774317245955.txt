Need to develop a personal website‑portfolio serving as an online resume for a specialist in system administration and DevOps.

#### 1. General Information

**1.1. Project Objective**  
- clearly showcase professional experience, skills, and achievements;  
- provide convenient access to contact information;  
- make a professional impression on recruiters and employers;  
- be SEO‑optimized and responsive across different devices.

**1.2. Target Audience**  
- HR specialists and recruiters at IT companies;  
- technical leaders (CTOs, IT department heads);  
- representatives of recruitment agencies specializing in IT personnel.

**1.3. Platform Requirements**  
- cross‑browser compatibility (Chrome, Firefox, Safari, Edge — latest versions);  
- responsive design (mobile devices, tablets, desktops);  
- support for high‑DPI screens (Retina and equivalents).

#### 2. Functional Requirements

**2.1. Main Website Sections**  
The website must include the following mandatory sections (tabs/blocks):

1. **Home**  
   - brief greeting and professional positioning (equivalent to the “Objective” section in a resume);  
   - specialist’s photo (optional);  
   - “Download Resume” button/link (PDF).

2. **About Me**  
   - short bio (1–2 paragraphs);  
   - key professional values and priorities;  
   - willingness to relocate/travel for work (if applicable).

3. **Work Experience**  
   - chronological list of employment (from most recent to oldest);  
   for each entry: company name, city, period, job title, 5–7 bullet points on responsibilities and achievements (with figures and results);  
   - option to collapse/expand details on click (for compactness on mobile devices).

4. **Skills (Technical Skills)**  
   - structured list of skills by category (OS, virtualization, cloud, monitoring, etc.);  
   - visualization of proficiency level (e.g., scale or icons).

5. **Education and Certifications**  
   - list of educational institutions, majors, graduation years;  
   - list of professional certifications with issue dates and numbers (if available).

6. **Achievements**  
   - 3–5 key professional achievements with measurable results (cost reduction, performance improvement, etc.).

7. **Contact**  
   - phone number;  
   - email;  
   - links to LinkedIn, GitHub, and other relevant profiles;  
   - contact form (optional).

8. **Blog/Articles** (optional)  
   - section for publishing technical articles, reviews, and case studies (if the specialist maintains a blog).

**2.2. Additional Features**  
- site search (if content volume is large);  
- dark/light theme (toggle);  
- scroll animations (smooth element appearance);  
- microdata (Schema.org) for SEO.

#### 3. Design and Interface

**3.1. Style**  
- minimalist, professional design;  
- color scheme: neutral tones (gray, blue, white) with accents on key elements;  
- fonts: readable sans‑serif (e.g., Roboto, Open Sans, Helvetica).

**3.2. Responsiveness**  
- mobile version: compact structure, vertical block layout;  
- tablet version: optimized spacing and font sizes;  
- desktop version: multi‑column layout for skills and experience sections.

**3.3. Interactivity**  
- smooth scrolling between sections;  
- tooltips on skill icon hover;  
- content loading animation (if AJAX is used).

#### 4. Technical Requirements

**4.1. Platform and Technologies**  
- **Frontend**: Vue.js (version 3.x) with Vite as the build tool.  
- **Project structure**: Composition API recommended; Single File Components (SFCs) mandatory.  
- **Styling**:  
  - Tailwind CSS (preferred) or vanilla CSS with PostCSS.  
  - Responsive design using Tailwind’s utility classes or CSS media queries.  
- **State management**: Pinia (official Vue store library) for global state (e.g., theme toggle, form data).  
- **Routing**: Vue Router for navigation between sections (if multi‑page layout is adopted).  
- **Backend**: Static site (no CMS). Vite will generate static assets.  
- **Hosting**: Any reliable provider supporting static files and HTTPS (e.g., Netlify, Vercel, GitHub Pages).  
- **Domain name**: Personal domain (e.g., `ivanov-sysadmin.com`).  

**4.2. Performance**  
- Image optimization:  
  - Convert to WebP format.  
  - Lazy loading for non‑critical images.  
  - Max width/height for avatars and icons (e.g., 200×200 px).  
- Code splitting: Vite should automatically split chunks for lazy‑loaded routes (if applicable).  
- Minification: Vite’s built‑in Terser plugin for JS/CSS.  
- Caching: Leverage browser caching via HTTP headers (configured on the hosting side).  

**4.3. Security**  
- HTTPS: Mandatory (SSL certificate provided by hosting platform).  
- XSS protection:  
  - Sanitize user input in the contact form (if implemented) using DOMPurify.  
  - Avoid `v-html` with untrusted content.  
- Form security:  
  - CAPTCHA (e.g., Google reCAPTCHA) for the contact form (if included).  
  - Rate limiting on form submissions (via backend proxy, if needed).  
- Regular backups: Hosting platform’s built‑in backup features.  

**4.4. SEO**  
- Meta tags:  
  - Dynamic `<title>` and `<meta name="description">` per page/section (managed via Vue Router meta fields).  
  - Open Graph tags for social sharing (e.g., LinkedIn previews).  
- Semantic markup:  
  - Proper use of `<h1>`–`<h6>`, `<article>`, `<section>`, etc.  
  - ARIA labels for interactive elements.  
- Sitemap: Generate `sitemap.xml` automatically via Vite plugin (e.g., `vite-plugin-sitemap`).  
- robots.txt: Include in the `/public` directory.  
- SSR/SSG: Not required (static site), but consider Nuxt.js in the future if SSR becomes necessary.  

#### 5. Content

**5.1. Content Sources**  
- text and data provided by the client (based on an existing resume);  
- images (specialist’s photo, skill icons) — provided by the client or selected from free stock resources (with license attribution).

**5.2. Text Requirements**  
- grammatical accuracy, no typos;  
- use of professional terminology;  
- brevity and clarity of formulations.
